# MeetScribe — Спецификация

## Обзор

Браузерное single-page приложение для захвата аудио вкладки (Google Meet, Zoom и т.п.) и транскрибации в реальном времени. Транскрипции хранятся **только локально** через File System Access API. Никакого сервера, никакого облака — GDPR-friendly.

**Ключевые ограничения дизайна:**
- Локальное хранение через File System Access API — один раз выбрал папку, handle сохраняется в IndexedDB
- Аудио уходит только транзитно в выбранный STT API
- Нет npm-зависимостей, нет сборщика, нет фреймворков
- Поддержка: Chrome 107+ / Edge 107+ на Linux и Windows (Firefox/Safari — нет)

---

## Технический стек

| Компонент | Решение |
|---|---|
| Фронтенд | Vanilla HTML + CSS + JS (`index.html` + `translations.js` + `audio-processor.js`) |
| Захват аудио вкладки | `getDisplayMedia({audio: true, video: true})` |
| Захват микрофона | `getUserMedia({audio: true})` |
| Микширование | Web Audio API (`AudioContext` 16kHz, `AudioWorkletNode`) |
| STT — Gemini | Gemini Live API (WebSocket), ASR side-channel, `gemini-2.5-flash-native-audio-latest` |
| STT — Deepgram | Deepgram Live WebSocket (`nova-2-general`), streaming PCM |
| STT — Groq | Groq REST API (Whisper Large v3 Turbo), буферизация + flush |
| STT — Web Speech | Web Speech API (`SpeechRecognition`), только микрофон |
| Запись аудио | `MediaRecorder` → WebM/Opus файл |
| Хранение файлов | File System Access API (`showDirectoryPicker`) |
| REST AI | Gemini / OpenAI / Mistral / Custom — для суммаризации, rename, чата, AI-транскрипции |
| Персистентность handle | IndexedDB |
| i18n | `translations.js` — EN/RU словари, `t(key)` helper |

## Функции

### F1 — Первый запуск / выбор папки

- При загрузке: проверить сохранённый `DirectoryHandle` в IndexedDB
- Если handle есть → `queryPermission({mode: 'readwrite'})`:
  - `granted` → использовать сразу, загрузить список сессий
  - `prompt` → показать баннер "🔓 Восстановить доступ"
- Если handle нет → онбординг-экран с кнопкой "📁 Выбрать папку"
- Handle сохраняется в IndexedDB (`meetscribe_dir_handle`)
- Без папки: кнопка "▶ Начать запись" недоступна (tooltip "Выберите папку")

### F2 — Настройки (⚙ кнопка)

Выдвижная панель (drawer) справа. Секции:

**Папка с транскрипциями** — показывает имя папки, кнопка "📁 Выбрать".

**Транскрипция в реальном времени:**
- Выбор движка: Gemini Live / Deepgram Live / Groq / Web Speech API
- Поле ключа для выбранного движка (скрывается для Web Speech)
- Порог эха (слайдер) — для определения говорящего
- Чекбокс "🎧 Наушники" — включает эхоподавление, меняет порог

**Summary API:**
- Выбор провайдера: Gemini / Mistral / OpenAI / Custom
- Поле ключа (сохраняется отдельно на каждый провайдер)
- При Custom — поле URL

**Deepgram (AI транскрипция файлов):**
- Отдельный ключ Deepgram для постфактум-транскрипции (🎙 кнопка)
- Если не задан — используется Gemini (fallback)

**Фон** — выбор фонового изображения из папки `backgrounds/` (скрывается если папка пустая).

**Тема** — авто / тёмная / светлая.

**Резервная копия:**
- "📦 Создать бэкап" — сохраняет все ключи, настройки, глоссарий в `.json` в папку транскрипций и скачивает на ПК
- "📂 Восстановить из файла" — восстанавливает настройки из `.json` файла

Все ключи сохраняются в `localStorage`. `DirectoryHandle` — в IndexedDB.

### F3 — Глоссарий (📖 кнопка)

Всплывающая панель. Поле для ввода терминов (имена, продукты, аббревиатуры) через запятую или по одному на строку. Сохраняется в `localStorage`. При старте записи термины добавляются в system instruction Gemini Live / Deepgram keyterm.

### F4 — Начало записи (▶ Начать запись)

Hotkey: `Alt+R`

1. Показывается capture hint — инструкция как выбрать вкладку в диалоге браузера
2. `getDisplayMedia({audio: true, video: true})` → пользователь выбирает вкладку Chrome
3. Если нет аудио трека / нет галочки "Поделиться звуком" → ошибка
4. `getUserMedia({audio: true})` → микрофон
5. `AudioContext({ sampleRate: 16000 })`, оба потока микшируются через AudioWorklet
6. Создаётся папка сессии `YYYY-MM-DD_HH-MM/` и `.txt` файл с хедером
7. `MediaRecorder` стартует → `.webm` файл
8. STT соединение открывается (WebSocket или REST в зависимости от движка)
9. Запускается energy interval (100ms) для определения говорящего и watchdog

Кнопки: "▶ Начать запись" → "⏹ Завершить" + "⏭ Завершить и новую" + "🎙 Мьют мик"

### F5 — Live-транскрипция (STT движки)

**Gemini Live** (`gemini-2.5-flash-native-audio-latest`):
- WebSocket к Gemini BidiGenerateContent API
- Setup: `inputAudioTranscription: {}` на уровне `setup` (не в `generationConfig`), `responseModalities: ['AUDIO']`
- Частичный текст → живая строка в UI (`S.liveBuf`)
- `turnComplete` → фиксируется реплика, дописывается в файл
- Автореконнект при обрыве (до 3 попыток)
- Watchdog: если сервер молчит >15 секунд при наличии звука — принудительный реконнект

**Deepgram Live** (`nova-2-general`):
- WebSocket к `wss://api.deepgram.com/v1/listen`
- Параметры: `encoding=linear16&sample_rate=16000&channels=1&punctuate=true&interim_results=true`
- `is_final=true` → фиксируется реплика

**Groq / Whisper** (Whisper Large v3 Turbo):
- Буферизация PCM чанков (~3s тишины или max 30s)
- При flush: PCM → WAV → POST `/openai/v1/audio/transcriptions`
- Реплики фиксируются по завершению запроса

**Web Speech API**:
- `SpeechRecognition` браузера, только микрофон
- `interimResults: true` → живая строка
- `onresult` с `isFinal=true` → фиксируется реплика
- ⚠️ Голоса других участников не транскрибируются

Формат реплики в файле: `[HH:MM:SS] Speaker: Текст` или `[HH:MM:SS] You: Текст`

### F6 — Определение говорящего

Energy-based: накапливает RMS энергию вкладки и микрофона за время реплики.
- Если `micTurnEnergy / total >= threshold` И `micPeakEnergy >= 0.015` → `You`
- Иначе → `Speaker`
- Peak floor 0.015 предотвращает ложные "You" от фонового шума
- Порог эха настраивается в настройках (слайдер)
- Headphones mode меняет порог (ниже = чувствительнее к микрофону)

### F7 — Мьют микрофона (🎙 кнопка)

Hotkey: `Alt+M`

Отключает/включает дорожку микрофона. Иконка меняется. Во время мьюта микрофонная энергия не учитывается при определении говорящего.

### F8 — Завершение записи (⏹ Завершить)

Hotkey: `Alt+S`

1. Флашим `liveBuf` — дописываем незавершённую реплику
2. Закрываем STT соединение
3. Останавливаем MediaStream треки
4. Дописываем footer: `---\nEnd of transcript\nDuration: HH:MM:SS`
5. Закрываем `FileSystemWritableFileStream`
6. Останавливаем `MediaRecorder` → `.webm` закрывается
7. Показываем тост AI-rename (F9)
8. Обновляем список сессий

### F9 — AI-rename после записи

Сразу после завершения: тост с полем ввода + спиннер.
Параллельно отправляется сниппет первых 30 строк транскрипции в REST API.
Промт требует slug-имя отражающее тему (не формат), без дат, только латиница/транслит.

Результат: папка `2026-04-15_14-31/` → `2026-04-15_14-31_product-vision/`
Все файлы внутри переименовываются каскадно (copy+delete, нативного rename в File System Access API нет).
Пользователь может отредактировать предложенное имя вручную или пропустить.

### F10 — Быстрый рестарт (⏭ Завершить и новую)

Hotkey: `Alt+N`

1. Завершает текущую сессию (F8)
2. AI-rename уходит в фоновый тост
3. Сразу стартует новая запись (F4)

### F11 — Переподключение вкладки

Если захватываемая вкладка была закрыта — показывается тост с кнопками:
- "🔄 Переподключить" — показывает capture hint, вызывает `getDisplayMedia` заново, подключает новый поток к существующему AudioWorklet без перезапуска записи
- "⏹ Завершить" — останавливает запись

### F12 — Список сессий

Боковая панель (растягиваемая перетаскиванием разделителя). Hotkey поиска: `Alt+F`

- Сортировка: новые сверху
- Группировка по дням: **Сегодня / Вчера / DD Month YYYY**
- `_dg.txt`, `_ai.txt`, `_chat.json` **скрыты** — группируются с базовым `.txt`
- В метаинфо: время, размер `.webm`, ` · DG` или ` · AI` если AI-транскрипция есть
- Поиск по имени файла (строка поиска)
- Кнопка ↻ — обновить список

Кнопки действий на каждой записи:
- **✨** — суммаризация (F13)
- **💬** — открыть чат (F15)
- **🎙** — AI-транскрипция (F16) — скрывается если AI-файл уже создан
- **🖼** — анализ экранов (F17)
- **⋯** — контекстное меню

Контекстное меню (⋯):
- 💾 Скачать — скачивает `.txt` файл
- ✏️ Переименовать — тост с AI-предложением и полем ввода (F9-логика)
- 📄 Оригинальная ASR — открывает базовый `.txt` (показывается только если есть AI-файл)
- 🗑 Удалить всё — удаляет всю папку сессии с подтверждением

Клик по записи — открывает AI-транскрипцию (если есть) или ASR в оверлее просмотра.

### F13 — Суммаризация (✨)

По нажатию ✨ — выпадающий список:
- Промты из папки `prompts/` в корне директории транскрипций (`.txt` / `.md` файлы)
- Если папки нет — встроенный промт (создание краткого структурированного отчёта)
- Выбор модели REST API

`{{transcript}}` в тексте промта заменяется содержимым файла. Если плейсхолдер отсутствует — транскрипция добавляется в конец промта через `\n\n`.

Результат (markdown) prepend-ится в начало файла с хедером:
```
# Meeting report
**Generated:** 24.04.2026 14:31
**Model:** Gemini / gemini-2.5-flash

---

{отчёт}

---

MeetScribe Transcript
...
```

Пишет в `aiFile` (`_dg.txt` или `_ai.txt`) если есть, иначе в базовый `.txt`.
При повторной суммаризации — удаляет старую преамбулу (до строки начинающейся с `MeetScribe`).

### F14 — Просмотр файла (оверлей)

Открывается кликом по сессии в списке. Рендеринг:
- `.md` файлы — markdown
- `.txt` файлы — "смешанный": верхняя часть (до первого `[HH:MM:SS]`) как markdown, транскрипт — как пузыри с временными метками
- Скриншоты из папки `screenshots/` — встроены как изображения с кнопкой удаления
- Кнопка фокус-режима — расширяет оверлей на весь экран
- Кнопка "▶ Содержимое" — сворачивает/разворачивает markdown-секцию

### F15 — Чат с транскриптом (💬)

Боковая панель, скользит поверх просмотра транскрипта.

- Контекст сессии (транскрипция) показывается вверху, сворачивается
- Быстрые вопросы (4 preset-кнопки)
- Поле ввода + кнопка отправки (Enter)
- Выбор промта из папки `prompts/` (кнопка ✨ рядом с полем)
- Выбор модели (кнопка рядом с ✨)
- История чата сохраняется в `_chat.json` внутри папки сессии
- Кнопка "🗑 Очистить чат" — удаляет `_chat.json`

Защита от race condition: контекст чата снимается snapshot в момент отправки — ответ всегда идёт в правильный файл даже если пользователь переключился на другую сессию.

Нормализация ролей для Gemini API: подряд идущие сообщения одной роли мержатся.

### F16 — AI-транскрипция файла (🎙)

Постфактум-транскрипция `.webm` аудио файла. Скрывается если AI-файл уже создан.

**Deepgram** (если задан `meetscribe_deepgram_key`):
- POST на `api.deepgram.com/v1/listen` с параметрами `model=nova-3&diarize=true&punctuate=true&keyterm=...`
- Параметр `keyterm` (не `keywords` — nova-3 не поддерживает)
- Результат с диаризацией (Speaker 0/1/2...) → `_dg.txt`
- Опциональная AI-идентификация спикеров по именам из транскрипта

**Gemini AI** (fallback):
- Загрузка `.webm` в Gemini Files API → ожидание ACTIVE статуса (до 4 минут)
- Транскрипция через `generateContent` с inline prompt
- Кэширование URI файла в localStorage (47 часов)
- Результат → `_ai.txt`

Оба варианта: тост с прогрессом, кнопки Отмена / Повторить.

### F17 — Анализ экранов (🖼)

Анализирует скриншоты из папки `screenshots/` внутри папки сессии.
Скриншоты делаются пользователем вручную во время записи (помещаются в папку).
Маркеры `[SCREENSHOT: filename.jpg]` должны быть в транскрипте (вставляются автоматически если скриншоты сделаны в нужный момент).

Процесс:
1. Находит все `[SCREENSHOT: fname]` маркеры в `.txt` файле
2. Для каждого скриншота — ищет ±15 секунд транскрипта для контекста
3. Отправляет изображение + контекст в vision API (Gemini / OpenAI / Mistral pixtral)
4. Вставляет описание обратно в `.txt` файл под маркером

Тост с прогрессом `🖼 0/N`. Кнопка Отмена.

## Аудио pipeline

```
[Вкладка] ──getDisplayMedia──► AudioContext 16kHz
[Микрофон] ──getUserMedia──►   AudioContext 16kHz
                                     │
                              AudioWorkletNode
                              (audio-processor.js)
                              Float32 → Int16 PCM
                                     │
                    ┌────────────────┴────────────────────┐
                    ▼                                     ▼
            STT WebSocket/REST                    MediaRecorder
            (Gemini/DG/Groq/WS)              createMediaStreamDestination()
                    │                                     │
            транскрипт .txt                         аудио .webm
```

- `AudioContext({ sampleRate: 16000 })` — единый контекст для STT и записи
- Запись: `bestAudioMime()` → `audio/webm;codecs=opus` → `audio/webm` → `audio/ogg;codecs=opus`
- PCM chunk: ~256ms, кодируется в base64 для WebSocket

## Обработка ошибок

| Ситуация | Поведение |
|---|---|
| Нет папки | Онбординг-экран, запись недоступна |
| Нет ключа Live API | Предупреждение при старте |
| Нет галочки "Поделиться звуком" | Сообщение об ошибке |
| Вкладка закрылась во время записи | Тост с кнопками "Переподключить" / "Завершить" |
| 429 / 503 от REST API | Retry: 0 → 5s → 12s → 25s (4 попытки) |
| Обрыв Gemini WebSocket | Автореконнект × 3, затем ошибка |
| Gemini zombie socket | Watchdog: >15s молчания при наличии звука → принудительный реконнект |
| Firefox / Safari | Сообщение "Требуется Chrome 107+" |
| Кастомный промт без {{transcript}} | Транскрипция добавляется в конец промта |

## Интернационализация

Две локали: **EN** (по умолчанию) и **RU**. Переключение кнопкой `EN/RU` в хедере.
Язык сохраняется в `localStorage('meetscribe_lang')`.
AI-промты (rename, summarize, chat system prompt) намеренно на русском — для русскоязычного вывода.
